@Title Table with ASCII-Codes their depending figures

@Section The ASCII-Codes 0 to 255 and their depending figures

  For information, the ASCII table below is the table used in Western Europe.
  Characters from 0 to 127 are US-ASCII standard and characters from 128 to 255 are 
  ISO 8859-1, whose full name is ISO/IEC 8859-1, and which is often called Latin-1 or 
  Western Europe and is Windows-1252 compatible (CP-1252).

@Section Extended descriptions of the common uses of the control characters (ASCII characters 0 to 31)

@FormatIf HTML WEBSITE
  <table border bgcolor="#FFFFFF" width="86%">
    <tr>
    <td><b>ASCII</b></td>
    <td><b>Description</b></td>
    </tr>
    <tr bgcolor="#FFFFCC"><td>0</td><td>Null-character, typically (and especially in PureBasic) used to indicate the end of the string.</td></tr>
    <tr bgcolor="#FFFFFF"><td>1</td><td>Start of heading. Indicates the start of a block of data, or the header portion of a block of data.</td></tr>
    <tr bgcolor="#FFFFCC"><td>2</td><td>Start of text. Typically sent as the first character in a block of text, during communications.</td></tr>
    <tr bgcolor="#FFFFFF"><td>3</td><td>End of text. Typically sent as the last character in a block of text, during communications.</td></tr>
    <tr bgcolor="#FFFFCC"><td>4</td><td>End of heading. Used to indicate the end of a transmission.</td></tr>
    <tr bgcolor="#FFFFFF"><td>5</td><td>Enquiry. Sent to a receiver in order to get a response.</td></tr>
    <tr bgcolor="#FFFFCC"><td>6</td><td>Acknowledge. Sent by a receiver to show that it has received and/or understood the request.</td></tr>
    <tr bgcolor="#FFFFFF"><td>7</td><td>Bell. Makes an audible noise (typically makes the PC speaker 'beep')</td></tr>
    <tr bgcolor="#FFFFCC"><td>8</td><td>Backspace. Moves the cursor one position to the left (might also delete the character to the left before the move)</td></tr>
    <tr bgcolor="#FFFFFF"><td>9</td><td>Tab. Horizontal tab, typically used to format text based tables.</td></tr>
    <tr bgcolor="#FFFFCC"><td>10</td><td>Linefeed. The character used to represent the function of moving forward one line on a typewriter or text mode printer. Typically used as the, or part of the, end of line characters.</td></tr>
    <tr bgcolor="#FFFFFF"><td>11</td><td>Vertical tab. Same as (horizontal) tab, but moves down one row rather than across one column.</td></tr>
    <tr bgcolor="#FFFFCC"><td>12</td><td>Form feed. Typically used as the character to tell a (text mode) printer to move onto the next page (form)</td></tr>
    <tr bgcolor="#FFFFFF"><td>13</td><td>Carriage return. The character which represents moving the head of a typewriter or printer back to the start of the line. Typically used as the, or part of the, end of line characters.</td></tr>
    <tr bgcolor="#FFFFCC"><td>14</td><td>Shift out. Start of a block of characters whose meaning is implementation dependant.</td></tr>
    <tr bgcolor="#FFFFFF"><td>15</td><td>Shift in. Closes the transmission of the above type of block.</td></tr>
    <tr bgcolor="#FFFFCC"><td>16</td><td>Data Link Escape. Used to indicate that the following control character should be interpreted as data rather than a control character.</td></tr>
    <tr bgcolor="#FFFFFF"><td>17</td><td>Device control 1. Typically used to turn on a piece of equipment. The most common use today is as the XON character in software flow controlled serial communications.</td></tr>
    <tr bgcolor="#FFFFCC"><td>18</td><td>Device control 2. Another situation-dependant device control character.</td></tr>
    <tr bgcolor="#FFFFFF"><td>19</td><td>Device control 3. Typically used to turn off a piece of equipment. The most common use today is as the XOFF character in software flow controlled serial communications.</td></tr>
    <tr bgcolor="#FFFFCC"><td>20</td><td>Device control 4. Another situation-dependant device control character.</td></tr>
    <tr bgcolor="#FFFFFF"><td>21</td><td>Negative acknowledge. Typically used to indicate not-received or not understood (errored) data.</td></tr>
    <tr bgcolor="#FFFFCC"><td>22</td><td>Synchronous idle. As the name implies, a signal which was sent synchronously (at periodic times) to indicate that the communications channel is idle, but still active.</td></tr>
    <tr bgcolor="#FFFFFF"><td>23</td><td>End of transmission block. Used to control transmission of data by indicating an end of block. This is not the same as EOT.</td></tr>
    <tr bgcolor="#FFFFCC"><td>24</td><td>Cancel. Commonly means that previously sent data should be ignored, although specifics are application dependant.</td></tr>
    <tr bgcolor="#FFFFFF"><td>25</td><td>End of medium. Used to indicate the end of some media, for example the end of a tape drive</td></tr>
    <tr bgcolor="#FFFFCC"><td>26</td><td>Substitute. A placeholder character used to indicate that a character has been substituted.</td></tr>
    <tr bgcolor="#FFFFFF"><td>27</td><td>Escape character. Usually the character produced by pressing the 'Esc' key on your keyboard, used in &quot;escape sequences&quot; to provide formatting information to text based displays (consoles, printers, etc)</td></tr>
    <tr bgcolor="#FFFFCC"><td>28</td><td>File separator</td></tr>
    <tr bgcolor="#FFFFFF"><td>29</td><td>Group separator</td></tr>
    <tr bgcolor="#FFFFCC"><td>30</td><td>Record separator</td></tr>
    <tr bgcolor="#FFFFFF"><td>31</td><td>Unit separator</td></tr>
  </table>
